Overview

This project uses Microsoft Threat Modeling Tool to diagram an Azure web application and analyze threats with STRIDE. The model includes Azure Front Door, App Service, API Management, Microsoft Entra ID, SQL Database, Blob Storage, Key Vault, and Monitor with clear trust boundaries.

Key outcomes

  • DFD with Azure boundaries
  • Auto generated threats
  • Mitigations mapped to Azure controls

Findings and mitigations

  • Use Azure Key Vault with managed identity for secret retrieval
  • Enforce HTTPS and min TLS 1.2 for all flows
  • Use private endpoints for SQL and Storage where applicable
  • Enable WAF on Front Door and validation policies on API Management
